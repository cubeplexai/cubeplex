# 成本跟踪

CubePlex 会跟踪整个组织的 LLM token 用量，帮助你监控支出并识别成本趋势。成本跟踪仪表板位于 **Admin > Cost**（`/admin/cost`）。

## 跟踪内容

通过 CubePlex 发起的每次 LLM 调用都会记录：

- **输入 token**——发送给模型的 token 数量（提示词、系统指令、工具结果等）。
- **输出 token**——模型在响应中生成的 token 数量。
- **缓存读取 / 缓存写入 token**——从提示词缓存中读取或写入提示词缓存的 token，按各自费率计费（通常远低于新的输入 token）。
- **提供商和模型**——处理该调用的提供商和模型。
- **时间戳**——调用的开始和结束时间。

每次调用还会记录每个模型的价格费率（输入、输出、缓存读取和缓存写入），并将其作为**调用时的快照**保存；因此，即使之后在[模型管理](./models.md)中更改模型费率，历史成本仍保持准确。

成本跟踪还会记录**沙箱计算**事件，因此汇总数据不仅反映 LLM token 支出。

## 用量仪表板

仪表板可让你从宏观角度查看组织的 AI 支出。

### 随时间变化的支出

支出的时间序列图，可按**每日**或**每周**粒度查看。你可以按**工作区**、**模型**或**用户**拆分该序列。使用它来发现趋势——例如新增团队成员后的用量增长、大批量任务导致的峰值，或改用更便宜模型带来的影响。

### 细分

除了全组织汇总外，仪表板还会按多种方式细分所选时段内的支出：

- **按模型**——哪些模型产生了最多支出，以及是否可以在某些使用场景中采用更便宜的替代方案。
- **按工作区**——哪些团队或项目支出最多。
- **按用户**——每个人的用量。
- **按天**——所选时段内的每日总量。

每一行都会显示输入/输出和缓存 token 数量、调用次数及成本。

### 导出

你可以将原始成本数据导出为 CSV——可导出整个组织的数据，也可导出单个工作区的数据——以便在电子表格中分析或导入自己的报表系统。

## 成本管理建议

- **设置准确的成本费率。** 确保[模型管理](./models.md)中的输入和输出 token 费率反映实际的提供商定价。费率不准确会导致仪表板数据产生误导。
- **定期查看。** 每周检查仪表板，以便及早发现异常峰值。
- **为任务选择合适的模型。** 如果大部分支出来自用高成本模型处理简单任务，请考虑引导团队在这些场景中使用更轻量的模型。
- **使用按模型视图进行比较。** 切换模型或添加新模型后，按模型细分可以显示该变更是否带来了预期的成本影响。
