# 成员与角色

CubePlex 使用两级角色系统：**组织角色**控制对管理设置的访问权限，**工作区角色**控制成员在特定工作区内可执行的操作。

组织级成员管理位于 **Admin > Members**（`/admin/members`）。工作区级角色则在每个工作区的设置中管理。

![Admin 成员表格，显示所有者和成员角色控件](/img/admin/members-table.png)

## 组织角色

| 角色 | 权限 |
|---|---|
| **Owner** | 完全控制。可管理提供商、模型、成员、成本跟踪和组织设置。每个组织只能有一位所有者。所有者的角色不可更改，且无法移除所有者。 |
| **Admin** | 可管理提供商、模型、成员和其他管理设置。不能更改所有者的角色或移除所有者。 |
| **Member** | 使用被分配的工作区。无权访问管理设置。 |

角色层级为：**Owner > Admin > Member**。更高角色继承所有较低角色的权限。

## 添加成员

1. 前往 **Admin > Members**。
2. 点击 **Add Member**。
3. 输入此人的邮箱地址。
4. 选择组织角色（Admin 或 Member）。
5. 点击 **Add**。

该人员必须已有 CubePlex 账户——系统会将邮箱与现有账户匹配；如果不存在该邮箱对应的账户，添加将失败。请新成员先注册，再将他们添加到组织。添加后，他们会立即获得其组织角色授予的访问权限。

## 更改成员的组织角色

1. 前往 **Admin > Members**。
2. 在列表中找到该成员。
3. 点击其姓名旁的角色下拉菜单。
4. 选择新角色。

:::note
无法在成员列表中更改所有者的角色——所有者显示为 **Owner** 徽标而非角色下拉菜单，且没有 **Remove** 操作。每个组织恰好有一位所有者。
:::

## 移除成员

1. 前往 **Admin > Members**。
2. 找到该成员并点击 **Remove**。
3. 确认移除。

移除成员会撤销其对组织中所有工作区的访问权限。其过去的对话历史会被保留。

## 工作区角色

在每个工作区中，成员可以拥有以下两种角色之一：

| 角色 | 权限 |
|---|---|
| **Workspace Admin** | 管理工作区设置、已安装工具、技能和成员访问权限。 |
| **Workspace Member** | 聊天、使用工具并与代理交互。 |

工作区角色应在工作区自身的设置页面中管理，而不是在组织级管理面板中。组织管理员可以访问任意工作区的设置。

## 常见场景

### 授予某人管理员权限，但不将其设为组织所有者

分配 **Admin** 组织角色。他们可以访问所有管理设置（模型、成员、连接器等），但无法更改或移除所有者。

### 将成员限制在特定工作区

分配 **Member** 组织角色，然后只将其添加到所需工作区。他们不会看到自己不是成员的工作区。
