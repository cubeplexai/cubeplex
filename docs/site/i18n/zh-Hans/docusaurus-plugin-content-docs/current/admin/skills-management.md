# 技能管理

技能将知识、行为模式或工作流打包为可安装模块，从而扩展代理能够执行的操作。作为管理员，你可以在整个组织中管理技能——上传自定义技能、连接外部注册表并控制可用性。

技能管理位于 **Admin > Skills**（`/admin/skills`）和 **Admin > Skill Registries**（`/admin/skill-registries`）。

## 技能属性

每个技能都具有以下属性：

| 属性 | 说明 |
|---|---|
| **名称** | 向用户显示的名称。 |
| **描述** | 技能的功能，会在发现和安装时显示。 |
| **关键词** | 用于搜索和分类的标签。 |
| **版本** | 语义化版本字符串。 |
| **来源类型** | 技能来源：`preinstalled` 或 `uploaded`。从远程注册表导入的技能存储为 `uploaded`，并会额外记录其来源注册表。 |

## 技能来源

技能的存储来源只有两个值：`preinstalled` 或 `uploaded`：

- **Preinstalled**——随 CubePlex 一起提供的技能，默认可用。管理员可以为组织卸载预装技能；CubePlex 会记录该决定，以便在下一次系统更新时不再恢复它。
- **Uploaded**——由组织自行创建并直接上传的自定义技能。从远程注册表（例如 [skills.sh](https://skills.sh)）导入的技能也会存储为 `uploaded`，并保留对其导入注册表的引用。

## 上传自定义技能

1. 前往 **Admin > Skills**。
2. 点击 **Upload Skill**。
3. 选择一个 `.zip` bundle。其根目录必须包含一个 `SKILL.md`，且该文件的 frontmatter 必须包含 `name`、`version` 和 `description`。（每个文件最大 10 MB，整个 bundle 最大 50 MB。）技能的名称、描述和关键词会从该 frontmatter 中读取。
4. 上传。CubePlex 会验证 bundle 并发布技能。

该技能会被添加到组织目录中。若要让它在每个工作区中可用，请在组织范围内安装它（见下文“安装并控制可用性”）。工作区所有者和成员也可以从技能发现界面自行安装目录中的技能。

## 连接技能注册表

技能注册表是托管技能集合的外部服务器。连接注册表后，其技能可在组织内被发现。

1. 前往 **Admin > Skill Registries**。
2. 点击 **Add Registry**。
3. 选择 **Kind**：内置提供商（skills.sh 或 ClawHub），或通用 **remote** 注册表。对于通用远程注册表，还需要输入其 **URL**（例如 `https://registry.example.com`）；内置提供商会自动使用其已知的 Base URL。
4. 为注册表命名。
5. 设置 **Trust tier**——`official`、`community` 或 `untrusted`。这是应用于通过该注册表发现的技能的默认信任级别。（对于 skills.sh，来自已识别官方上游的单个技能会被提升为 `official`，不受此默认值影响。）
6. 点击 **Save**。

注册表连接并启用后，用户搜索时，其技能会出现在发现界面中。之后你可以在注册表详情面板中启用、禁用或更改其信任级别。

## 安装并控制可用性

在 **Admin > Skills** 中，你可以控制组织中有哪些技能，以及它们如何提供给工作区：

- 在组织范围内**安装**技能，使其可供每个工作区使用。每个组织范围的安装都带有 **auto-bind** 设置：开启时，技能会在工作区中自动启用；关闭时，工作区所有者必须在每个工作区中手动开启。预装技能默认开启 auto-bind；上传的技能默认关闭。
- **卸载**技能以将其从组织中移除。这也会移除其每工作区绑定。卸载*预装*技能会记录一个 tombstone，因此 CubePlex 不会在下一次系统更新中恢复它。

特定工作区能否使用组织已安装的技能，随后由每工作区开关决定（工作区所有者可在工作区 **Skills** 页面中管理，管理员也可在技能的工作区绑定视图中管理）。为某个工作区禁用技能会在该工作区中对代理隐藏它；历史对话输出不受影响。

## 管理技能版本

每次上传或从注册表导入都会追加一个新的不可变版本；旧版本永远不会被修改。当存在比组织当前安装版本更新的版本时，技能会显示 **update available**。

1. 前往 **Admin > Skills** 并选择该技能。
2. 在详情面板中查看可用版本。
3. 安装希望组织使用的版本。安装最新版本会获取更新；安装较早版本会将组织固定到该版本。
