# 核心概念

本页面介绍 CubePlex 的基本构建模块。理解这些概念能帮助你更充分地使用平台。

## 组织和工作区

CubePlex 采用层级结构：

**组织** 是顶层账户。它拥有账单、提供商 API 密钥和组织范围的策略。每个用户恰好属于一个组织。

**工作区** 位于组织内。每个工作区都是独立的协作空间，拥有各自的对话、技能、记忆和 MCP 工具授权。一个用户可以是多个工作区的成员。

**角色** 决定你可以执行哪些操作：

| 范围 | 角色 | 可以执行的操作 |
|---|---|---|
| 组织 | 所有者、管理员、成员 | 所有者/管理员：管理提供商、模型、成员和组织设置。成员：使用已分配的工作区。 |
| 工作区 | 管理员、成员 | 管理员：管理工作区设置、工具和技能。成员：聊天、使用工具。 |

## 对话

对话是你与 AI 智能体之间的消息线程。每个对话都关联到特定模型（你可以在对话过程中更换模型）。智能体可以：

- 以 **文本** 和 **思考**（扩展推理）形式回复。
- 通过 MCP 连接器对外部服务发起 **工具调用**。
- 在沙箱环境中执行 **代码**。
- 生成 **制品**——文件、网站、图片或其他可交付内容。

你可以在消息中附加文档、图片和代码文件。对话会自动保存，并显示在侧边栏历史记录中。

## 技能

技能是你安装后可扩展智能体能力的打包功能。可以将其视为向智能体提供新知识或行为模式的插件。

**技能的三种来源：**

- **内置技能** — 随 CubePlex 一同提供。
- **组织上传** — 由你的组织创建和共享的自定义技能。
- **远程注册表** — 托管在 [skills.sh](https://skills.sh) 等注册表中的社区技能。

你可以在对话中，或从工作区侧边栏的 **技能** 页面发现并安装技能。安装后，只要技能与对话相关，智能体便可使用它。

详情请参阅[技能指南](../guides/skills/overview.md)。

## 记忆

记忆让智能体能够跨对话记住信息。CubePlex 使用三级系统：

| 层级 | 可见范围 | 使用示例 |
|---|---|---|
| **个人** | 仅自己可见，覆盖你的所有工作区 | “我更喜欢 Python 而不是 JavaScript” |
| **工作区** | 工作区的所有成员 | “我们的 API 对 JSON 键使用 snake_case” |
| **组织** | 所有工作区中的全部成员 | “公司名称为 Acme Corp，成立于 2020 年” |

**记忆类型** 描述所记住的信息：

- **preference** — 你偏好的做事方式。
- **project_fact** — 有关项目或代码库的事实。
- **procedure** — 分步骤的流程。
- **correction** — 智能体出错后由你纠正的内容。
- **decision** — 已达成一致的决策。
- **org_policy** — 组织范围的规则。

智能体会在每次对话开始时自动读取相关记忆。你也可以手动查看、编辑和删除记忆。

详情请参阅[记忆指南](../guides/memory/overview.md)。

## MCP 工具

模型上下文协议（Model Context Protocol，MCP）连接器让智能体能够调用外部 API，包括数据库、SaaS 产品、内部服务等。

**工具生命周期的运作方式：**

1. **模板** — 可用连接器的目录（例如“GitHub”“Slack”“PostgreSQL”）。模板位于目录中，其可见范围可以是全局（所有组织）、按组织或按工作区。
2. **启用** — 工作区管理员（或通过分发操作的组织管理员）为工作区启用模板，使其可配置凭据。
3. **凭据** — 工具使用者根据连接器的认证模式连接组织级、工作区级或用户级凭据。
4. **运行时** — 已配置凭据的工具可供智能体在对话中使用。

**认证模式：**

- **静态凭据** — 你提供固定密钥（API 密钥或 bearer token）。连接器会根据其配置，通过 `Authorization` 标头、自定义标头或查询参数发送该密钥。
- **OAuth** — 连接器会引导你完成 OAuth 流程。
- **无** — 连接器不需要凭据。

详情请参阅[MCP 工具指南](../guides/mcp/overview.md)。

## 制品

制品是智能体在对话中生成的可交付内容。它们不只是纯文本——制品可以是可下载文件、实时网站预览、代码片段、图片或数据表。

制品具有以下特性：

- **可版本化** — 智能体可以在多个对话轮次中迭代制品。
- **可预览** — 可在对话中内联渲染。
- **可下载** — 将制品保存到本地计算机。

## 自动化

自动化让你无需手动操作即可运行智能体任务。

- **计划任务** — 按 cron 计划、固定间隔运行，或在特定时间一次性运行。
- **事件触发器** — Webhook 触发后，会将 webhook 载荷作为上下文启动一次智能体运行。

详情请参阅[自动化指南](../guides/automation/scheduled-tasks.md)。
