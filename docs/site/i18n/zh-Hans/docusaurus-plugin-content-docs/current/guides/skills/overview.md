# 技能概览

技能是可扩展智能体能力的打包功能。一个技能将说明（以 `SKILL.md` 文件形式提供）与可选的支持文件——脚本、模板、配置——打包为单个可安装单元。对话期间加载技能后，智能体会遵循其中的说明完成特定任务，例如生成演示文稿、起草电子邮件、分析数据集、使用特定框架编写代码等。

## 技能如何运作

1. **技能被安装** 到你的工作区中（或者由管理员在整个组织范围内安装）。
2. **智能体会看到它**，在该工作区中每次对话开始时，它都会列在智能体可用技能中。
3. **相关时，智能体会加载它** — 可能是因为你提出的请求属于该技能的范围，也可能是因为你明确要求使用该技能。加载会将技能说明注入智能体的上下文。
4. **智能体会遵循说明** 完成任务，并在沙箱中使用技能提供的任何支持文件（模板、脚本等）。

你无需记住技能名称。智能体会自动将你的意图与已安装技能进行匹配。例如，如果你说“创建一份关于第三季度业绩的演示文稿”，智能体会在有可用演示文稿技能时加载它。

## 技能来源

技能来自三个地方：

- **内置（预安装）** — 随 CubePlex 提供，默认可用。如果不需要，组织管理员可以禁用特定内置技能。
- **已上传** — 由组织管理员（或你在工作区级别）发布。这些是为团队工作流量身定制的自定义技能。
- **远程注册表** — 管理员已连接的外部技能源（如 [skills.sh](https://skills.sh)）。你可以搜索这些注册表，并直接从中安装技能。

## 信任等级

远程注册表中的技能带有信任指示，让你了解所安装的内容：

| 等级 | 含义 |
|---|---|
| **官方** | 已由注册表维护者或已知发布者审核。 |
| **社区** | 由社区贡献者发布；未经过正式审核。 |
| **不受信任** | 尚未进行任何审核。安装前请检查技能内容。 |

内置和已上传技能本身是可信的，因为它们来自 CubePlex 或你自己的组织。

## 安装范围

技能可以在两个级别安装：

- **组织范围** — 管理员为整个组织安装。它会在每个工作区中可用（管理员可在每个工作区中切换其启用状态）。
- **工作区私有** — 你将其安装到单个工作区。只有该工作区的成员可以使用它。

## 技能中包含什么

每个技能至少包含一个 `SKILL.md` 文件——其中包括 frontmatter（名称、版本、说明、关键词）以及智能体需要遵循的指令。许多技能还包含：

- **脚本** — 智能体可在沙箱中运行的 Shell 或 Python 脚本。
- **模板** — 起始文件、样板内容或参考资料。
- **配置** — 控制技能行为的设置。

技能加载后，其文件会挂载在沙箱内的 `/.skills/<name>/<version>/`，以便智能体引用。智能体会使用技能的准确名称调用内置 `load_skill` 工具来读取技能说明。
