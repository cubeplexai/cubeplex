# 记忆概览

记忆让 CubePlex 智能体能够跨对话记住信息。你无需每次开始新聊天时都重复说明，智能体会自动回忆你的偏好、项目事实和团队约定。

## 三级记忆

CubePlex 将记忆划分为三个范围。每个范围决定谁能查看并从某条记忆中受益。

| 层级 | 可见者 | 最适合的内容 |
|---|---|---|
| **个人** | 组织内任意工作区中的你自己 | 个人偏好、更正、个人笔记 |
| **工作区** | 工作区的所有成员 | 项目事实、团队流程、代码库约定 |
| **组织** | 所有工作区中的所有人 | 公司范围的政策、品牌指南、共享决策 |

智能体处理对话时，会从这三个层级中提取相关条目。提供给智能体的记忆块会按范围分组（先组织，再工作区，最后个人），并将工作区和组织条目标记为用户贡献内容，以便智能体适当地衡量它们。在任一单独范围内，`correction` 条目的优先级高于同一主题的普通记忆。

## 记忆类型

每条记忆都按类型分类，这有助于智能体理解如何应用它：

- **preference** — 你或团队偏好的做事方式。*“我更喜欢 TypeScript 而不是 JavaScript。”*
- **project_fact** — 有关项目、代码库或领域的具体事实。*“我们的 API 对所有 JSON 键使用 snake_case。”*
- **procedure** — 要遵循的分步骤流程。*“部署时，请先运行 `make build`，然后运行 `make deploy-staging`，再推送到生产环境。”*
- **correction** — 智能体出错后由你纠正的内容。*“是 PostgreSQL，不是 MySQL——我们去年已经迁移了。”*
- **decision** — 已达成一致且今后应遵循的决策。*“我们为新仪表板选择了 Tailwind，而不是 CSS modules。”*
- **org_policy** — 组织范围的规则或标准。*“所有公共 API 必须包含速率限制。”*

## 记忆如何运作

智能体会自动从两个方向处理记忆：

**回忆** — 在每个对话轮次开始时，智能体会检索与当前上下文相关的记忆条目。你无需要求它“检查记忆”——它会自行完成。

**存储** — 当你在对话中分享重要信息时，智能体可能会将其保存为记忆条目。发生这种情况的场景包括：

- 你明确提出要求：*“请记住，我们的预发布 URL 是 staging.example.com。”*
- 你纠正智能体：*“不，我们使用 Poetry，不使用 pip。”*
- 智能体在对话中识别出可复用的事实。

每条已存储的记忆都包括：

- **内容** — 实际信息。
- **范围** — 个人、工作区或组织。
- **类型** — 上述六种类型之一。
- **来源** — 指向创建该记忆的对话、运行或制品的引用。
- **置信度** — 由智能体自行评定的 0.0 至 1.0 分数，表示它对记忆准确性的信心程度。新条目的默认值为 0.8。
- **状态** — 活跃（用于回忆）或已归档（从回忆中隐藏但未删除）。
