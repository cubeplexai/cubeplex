# 管理记忆

记忆中心用于查看智能体记住的内容，并停用不再准确的条目。你可以浏览所有有权访问的范围内的记忆，并归档任何活跃条目。

## 打开记忆中心

进入工作区并打开 **记忆**。记忆中心位于 `/w/{workspaceId}/memory`，其中 `{workspaceId}` 是当前工作区。你也可以从对话中进入：当智能体保存或更新一条记忆时，它显示的标签会直接链接到记忆中心，并预先筛选为该对话。

![记忆中心，显示范围标签、置信度和已填充的记忆卡片](/img/memory/memory-center.png)

## 浏览记忆条目

记忆中心分为四个标签：

- **个人** — 限定给你的活跃记忆。
- **工作区** — 在当前工作区中共享的活跃记忆。
- **组织** — 在整个组织中共享的活跃记忆。
- **已归档** — 你已停用的、涵盖所有范围的条目。

每张卡片都会显示：

- **内容** — 已存储的信息。
- **类型** — 彩色徽章：偏好、事实、流程、更正、决策或组织政策。
- **置信度** — 智能体自评的置信度，以百分比显示（底层分数范围为 0.0 到 1.0）。
- **更新时间** — 条目上次更改的时间，以相对时间显示（“今天”“3 天前”）。
- **来源提示** — 对智能体在聊天期间创建的条目显示“来自对话”标记。
- **状态** — 已归档条目带有“已归档”徽章；活跃条目则没有。

## 筛选到单个对话

从对话的记忆标签打开记忆中心时，系统会仅筛选与该对话关联的条目，并在顶部显示横幅。点击横幅中的 **清除** 可返回完整列表。

## 编辑记忆

记忆中心仅支持查看和归档。要更改记忆条目的 *内容*，请在对话中请求智能体操作——例如，“将有关我们 API 基础 URL 的记忆更新为 `api.example.com`”。智能体会就地编辑现有条目，而不是创建重复项。更改条目的类型或置信度也是如此：这些更新应通过智能体完成，而不是通过记忆中心 UI。

## 归档

要归档活跃记忆，请将鼠标悬停在其卡片上，然后点击右上角的归档按钮。归档会将条目状态设为“已归档”：它不再参与智能体的回忆，但仍保留在系统中，并移至 **已归档** 标签。

记忆中心没有永久删除按钮——归档是将条目退出活跃使用的方式。如果你不再希望智能体回忆某条内容，请归档它。（在幕后，即使 API 中的删除操作也是软删除：它会归档条目，而不是将其彻底擦除。）

## 实用工作流

### 在项目转向后清理

如果团队更换技术栈或重命名项目，某些工作区记忆将会过时。打开 **工作区** 标签，检查不再适用的项目事实并将其归档。如需用更正后的内容替换，请在对话中请求智能体更新它。

### 查看智能体更正

打开 **工作区** 标签，查看“更正”徽章以了解智能体在哪些地方被纠正。这有助于发现模式——如果同一更正不断出现，请考虑让智能体保存更清晰的事实或流程，以从一开始就避免误解。

### 新团队成员入职

新成员会自动从工作区和组织记忆中受益，无需执行任何操作。如果你希望查看智能体会向他们提供哪些信息，请打开 **工作区** 和 **组织** 标签，查看所有共享知识。

## 谁可以管理哪些记忆

访问权限取决于记忆范围。在某个范围内，任何可以查看条目的用户也可以归档或更新它——记忆没有单独的“仅创建者”或“仅管理员”限制：

| 记忆范围 | 可查看者 | 可归档/更新者 |
|---|---|---|
| 个人 | 仅你自己 | 仅你自己 |
| 工作区 | 该工作区的所有成员 | 该工作区的所有成员 |
| 组织 | 该组织的所有成员 | 该组织的所有成员 |

## 提示

- **定期查看。** 数月前创建的记忆条目可能已不再准确。每月快速检查一次工作区标签，可让智能体保持敏锐。
- **归档即审计记录。** 已归档条目会保留在“已归档”标签中，作为智能体过去所知内容的记录，尽管它们不再影响回忆。
- **让智能体更新，而不是重复保存。** 如果事实发生变化，请让智能体更新现有记忆，而不是保存一条新的。智能体会就地编辑；重复项可能导致它回忆到相互冲突的信息。
