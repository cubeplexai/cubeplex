# 话题

**话题** 是一组相关对话的容器。它有参与者名单，并可以容纳多个相互独立的消息线程；每位话题成员都能查看并加入这些线程。

当团队需要围绕一个项目、调查或事故拆分多个并行工作流时，使用话题。话题不是群聊：群聊是一条共享对话，而话题是多个对话的容器（参阅[群聊](./group-chats.md)）。

## 话题与其他对话模型

| | 一对一对话 | 群聊 | 话题 |
|---|---|---|---|
| 结构 | 一条消息线程 | 一条消息线程 | 一个或多个对话的容器 |
| 参与者 | 你和智能体 | 一条对话上的成员名单 | 话题下所有对话共享的成员名单 |
| 侧边栏 | 一个独立条目 | 一个独立条目 | 一个父条目，下面嵌套多个对话 |
| 沙箱 | 你的个人沙箱 | 对话范围的共享沙箱 | 话题范围的共享沙箱 |

## 创建话题

话题有三种创建方式。无论使用哪种方式，创建者都会成为其 **所有者**。

### 新建话题

点击侧边栏中的 **新建话题**。输入标题、邀请工作区成员，并选择[沙箱模式](#共享沙箱)。CubePlex 会创建带有第一个对话的话题并将其打开。

![显示沙箱选项的新建话题对话框](/img/conversations/topic-create-dialog.png)

### 升级现有对话

当一对一对话需要拆分为多个线程时，在对话标题栏中使用 **升级为话题**。现有对话会成为新话题下的第一个对话，你将成为其所有者。

已属于话题，或已关联 IM 机器人、计划任务、事件触发器的对话不能升级。

### 从 IM 机器人创建

IM 连接器使用 **topic** 路由模式时，会在首次收到消息时创建 CubePlex 话题，并保存 IM 平台和频道元数据。使用 **shared** 路由模式时，连接器使用的是一条独立群聊，不会创建话题。参阅 [IM 连接器](../im/overview.md) 和[群聊](./group-chats.md)。

:::note Web 话题和 IM 话题无法互通（当前限制）
IM 运行器只能驱动源自 IM 的话题。你在 Web 应用中创建的话题目前无法通过 IM 频道获得回复。
:::

## 参与者和角色

每位参与者都具有以下两种角色之一：

- **所有者** — 管理参与者、标题和话题生命周期。
- **成员** — 可以阅读和发布话题下每个对话中的内容，并开始新对话。

| 操作 | 可执行者 |
|---|---|
| 邀请参与者 | 仅所有者 |
| 离开话题 | 任意参与者（自己） |
| 移除其他参与者 | 仅所有者 |
| 转移所有权/更改角色 | 仅所有者 |
| 重命名话题 | 仅所有者 |
| 置顶/取消置顶话题 | 任意参与者 |
| 归档话题 | 仅所有者 |
| 在话题中开始新对话 | 任意参与者 |

受邀者必须已经是工作区成员。话题仅对参与者可见。如果所有者离开或被移除，所有权会转给最早加入的剩余参与者。

默认参与者上限为 **20**；从共享 IM 频道创建的话题最多可容纳 **100** 位参与者。

## 共享沙箱 {#共享沙箱}

话题拥有一个共享的[代码执行沙箱](../../admin/sandbox.md)，创建时选择且之后不能更改：

- **专用话题沙箱** — 新建属于话题的沙箱，存储隔离；开始时所在对话中的文件不会带入。这是 **新建话题** 的默认选项。
- **复用创建者的沙箱** — 在所有者的个人沙箱中运行，保留现有文件和环境；所有成员的代码都会在该环境中执行。这是升级现有对话时的默认选项。

:::caution
复用模式下，其他成员的操作会在所有者的个人沙箱中执行。仅应与信任其使用该环境的人一起使用。
:::

可在工作区[沙箱](./sandboxes.md)设置标签中管理话题沙箱。

## 话题中的对话

话题从一个对话开始，任意参与者都可以继续创建对话，通常每个子任务一个。它们均对所有成员可见，并在侧边栏中嵌套在话题下方。话题没有额外的一条消息线程。

共享话题中的每条消息都会标示发送者；普通一对一聊天只有一个人类参与者，因此不会显示该标记。

## 排序、置顶和归档

- **排序。** 话题、群聊和一对一对话共用一个侧边栏列表，并按最近活动排序。话题中的任意对话有新消息时，话题会移动。
- **置顶。** 置顶话题对所有成员生效；置顶单个对话只会提升该对话。
- **归档。** 所有者可为所有成员归档话题。成员可从话题成员控件中离开。

## 将话题作为自动化目标

计划任务可以将话题作为目标，使定期结果发布到共享工作区，而不是私人对话。参阅[计划任务](../automation/scheduled-tasks.md)。

## 提示

- 有意识地选择沙箱模式；它无法在之后更改。
- 需要保留历史记录时，升级现有的一对一对话。
- 并行工作流使用多个对话；如果所有人应该留在同一个线程，请使用[群聊](./group-chats.md)。
