# 话题

**话题** 可将智能体变成多人共享的工作区。普通对话仅包含你和智能体；话题则有一份 **参与者名单**，并可在同一话题下包含 **多个对话**——话题中的每个人都能查看并加入其中的每个对话。

当多个人需要与同一智能体共同处理同一件事时，请使用话题：项目频道、团队调查、值班室，或由 IM 平台驱动的群聊。

## 话题与对话

| | 对话 | 话题 |
|---|---|---|
| 参与者 | 仅你自己 | 多人（你和受邀成员） |
| 包含内容 | 一个消息线程 | 一个或多个对话 |
| 侧边栏 | 独立的一行 | 群组节点，其对话嵌套在下方 |
| 沙箱 | 你的个人沙箱 | 共享沙箱（参阅[共享沙箱](#the-shared-sandbox)） |

只有一位参与者的话题在你添加其他成员前，其行为与普通一对一聊天相同。

## 创建话题

话题可以通过三种方式创建。无论使用哪种方式，创建者都会成为其 **所有者**。

### 新建话题

点击侧边栏中的 **新建话题**。在对话框中，为其命名、邀请工作区成员，并选择[沙箱模式](#the-shared-sandbox)。CubePlex 会创建带有第一个对话的话题，并将你带入其中。

### 升级现有对话

已经在一对一对话中深入讨论，而它应该变成团队协作？从对话标题栏中使用 **升级为话题**。现有对话会成为新话题下的第一个对话，你将成为其所有者。

你无法升级已属于某个话题的对话，也无法升级已关联 IM 机器人、计划任务或事件触发器的对话——请先解除这些关联。

### 从 IM 机器人创建

当机器人以 **topic** 或 **shared** 路由模式绑定到群聊时，CubePlex 会在该聊天中首次收到消息时自动创建话题。这些由 IM 创建的话题会将来源平台和频道作为元数据保存。参阅 [IM 连接器](../im/overview.md)。

:::note Web 话题和 IM 话题无法互通（当前限制）
IM 运行器只能驱动源自 IM 的话题。你在 Web 应用中创建的话题目前无法通过 IM 频道获得回复。
:::

## 参与者和角色

每位参与者都具有以下两种角色之一：

- **所有者** — 创建者。管理参与者名单、标题和话题的生命周期。
- **成员** — 可以阅读和发布话题下每个对话中的内容，并在其中开始新对话。

| 操作 | 可执行者 |
|---|---|
| 邀请参与者 | 仅所有者 |
| 离开话题 | 任意参与者（自己） |
| 移除其他参与者 | 仅所有者 |
| 转移所有权/更改角色 | 仅所有者 |
| 重命名话题 | 仅所有者 |
| 置顶/取消置顶话题 | 任意参与者 |
| 归档话题 | 仅所有者 |
| 在话题中开始新对话 | 任意参与者 |

受邀者必须已经是该工作区的成员——邀请他们加入话题不会将其添加到工作区，也不会创建账户。话题仅对参与者可见；工作区中的其他人无法看到它。

如果所有者离开或被移除，所有权会自动转给最早加入的剩余参与者，因此话题永远不会没有所有者。

**参与者上限：** 默认情况下，一个话题最多可容纳 **20** 位参与者。从共享 IM 频道创建的话题最多可容纳 **100** 位参与者。

## 共享沙箱 {#the-shared-sandbox}

由于话题是共享的，其[代码执行沙箱](../../admin/sandbox.md)也是共享的。创建话题时需要选择其运行方式，且 **之后无法更改**——请慎重选择：

- **专用话题沙箱** — CubePlex 会启动一个 **属于话题的全新沙箱**。它拥有独立存储，与任何人的个人沙箱和所有其他话题相互隔离。你开始时所在对话中的文件 **不会** 被带入。它是 **新建话题** 的默认选项，也是更适合群组的选择。
- **复用创建者的沙箱** — 话题会在 **所有者的个人沙箱** 中运行。文件和环境会被保留，当你升级自己的对话并希望保留其工作文件时会很方便。但代价是：**每位参与者的代码都会在所有者的环境中运行**，并可访问该环境中的所有内容。它是 **升级** 现有对话时的默认选项。

:::caution
在“复用创建者的沙箱”模式下，其他成员的操作会在你的个人沙箱中执行。仅应与那些你愿意信任其使用自己环境的人共同使用此模式。
:::

你可以在工作区[沙箱](./sandboxes.md)设置标签中查看和管理你拥有的所有沙箱，包括话题的专用沙箱。

## 话题中的对话

话题开始时包含一个对话，但任何参与者都可以打开更多对话——例如，每个子任务一个对话。它们均对所有参与者可见，并在侧边栏中嵌套在话题下方。没有单独的“话题页面”；你可以像使用任何其他聊天一样在话题的对话中工作。

**每条消息的发送者。** 在共享话题中，每条消息都会标示发送它的参与者，因此你可以一眼区分贡献者；智能体也会获知是谁在说话，以便跟踪上下文。在普通一对一聊天中，此标签会隐藏——只有你自己。当你将一对一对话 **升级** 为话题后，你此后发送的消息会带上你的名称，新参与者即可看到每句话的发送者。

## 排序、置顶和归档

- **排序。** 话题和独立对话共用一个侧边栏列表，并按最近活动排序。话题内的任意对话收到新消息时，该话题都会上浮。
- **置顶。** 任意参与者均可置顶话题，使其保持在顶部。置顶是共享的——它会影响所有人看到的话题位置。（置顶单个对话只会将该对话置于顶部，即使它属于某个话题。）
- **归档。** 所有者可从侧边栏菜单中归档话题。归档的话题会从所有参与者的侧边栏中消失。只想退出的个别成员可以从成员面板中 **离开** 话题。

## 将话题作为自动化目标

计划任务可以将话题作为目标，从而让定期运行的结果发布到共享群组，而不是私人对话。参阅[计划任务](../automation/scheduled-tasks.md)。

## 提示

- **有意识地选择沙箱模式。** 这是唯一无法在以后更改的话题设置。除非确实需要创建者的文件，否则默认选择专用沙箱。
- **升级，不要重建。** 如果一对一聊天发展为团队协作，请升级它——这样可以保留历史记录和（可选的）工作文件。
- **为不同线程使用不同对话。** 一个话题中包含多个对话能让群组的并行工作流保持清晰，而不会交织在单个线程中。
