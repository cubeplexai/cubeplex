# 群聊

**群聊** 是一条由多位工作区成员共享的对话。它保留一份消息历史，并使用对话范围的共享沙箱。它与[话题](./topics.md)是不同概念：话题是可以容纳多个对话的容器。

## 何时使用群聊

当所有人都应该留在同一个线程中时，使用群聊：快速评审、交接，或围绕当前对话文件的短期讨论。当团队需要在同一个项目或事故下拆分多个线程时，使用话题。

| | 一对一对话 | 群聊 | 话题 |
|---|---|---|---|
| 消息历史 | 一条私有线程 | 一条共享线程 | 一个容器下的多条线程 |
| 成员关系 | 一位人类参与者 | 对话上的成员名单 | 所有子对话共享的成员名单 |
| 侧边栏 | 独立对话条目 | 带群聊状态的独立条目 | 一个父条目，下面嵌套多个对话 |
| 沙箱范围 | 个人 | 对话范围，共享 | 话题范围，共享 |

## 将对话变成群聊

1. 打开要共享的对话。
2. 在对话标题栏中选择 **邀请**。
3. 选择一位或多位已有工作区成员，并确认 **邀请**。

第一次邀请后，原有对话会留在原处并变成群聊。消息历史会保留，参与者可以在同一个线程中查看和回复。

![邀请加入此聊天对话框](/img/conversations/group-chat-invite.png)

只有工作区成员可以被邀请。邀请不会将对方添加到工作区，也不会创建账户。

## 共享沙箱与记忆

一对一对话变成群聊后：

- 对话的沙箱会与参与者共享；
- 参与者可以查看和修改该沙箱中的文件；
- 智能体运行时不再注入个人记忆。

这是对话级别的变化。它不会创建话题，也不会创建额外对话。如果团队需要新的隔离环境和多个工作流，请改用话题。

:::caution
只邀请你信任其使用当前对话沙箱的人。现有文件和环境设置会继续对群聊可用。
:::

## 群聊与话题

判断标准很简单：

- **一条线程、一份历史：** 群聊。
- **多条线程、一份共享成员名单：** 话题。

IM 连接器的 **shared** 路由模式对应前者；**topic** 路由模式对应后者。参阅 [IM 连接器](../im/overview.md)。

## 提示

- 从要保留的对话中邀请成员；现有历史会保留。
- 并行工作请使用话题，不要创建互不相关的多个群聊。
- 把共享沙箱当作协作状态：每位参与者都可以修改其中的文件。
