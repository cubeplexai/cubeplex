# 附件

你可以在消息中附加文件，以便智能体读取、分析或引用它们。附件仅属于上传它们的对话。

## 支持的文件类型

CubePlex 将附件分为三类：

| 类型 | 示例 |
|---|---|
| **图片** | PNG、JPEG、WebP、GIF |
| **文档** | PDF、纯文本、Markdown、CSV、DOCX、XLSX、JSON、YAML |
| **其他** | 组织配置允许的任何其他文件类型 |

允许的 MIME 类型具体由组织管理员配置。如果你尝试上传不被允许的文件类型，上传将被拒绝并显示错误。

## 如何附加文件

### 点击上传

1. 点击输入栏中的 **回形针图标**。
2. 从文件选择器中选择一个或多个文件。
3. 文件上传时会以标签形式显示在输入区域上方。
4. 输入消息（可选），然后按 **Enter** 发送。

你可以在一条消息中附加多个文件，但不能超过每条消息的上限（参阅[大小和配额限制](#size-and-quota-limits)）。

### 拖放

将文件从桌面拖到聊天区域。系统会显示拖放区域覆盖层，确认放置目标。松开文件后，上传会立即开始。

### 发送前移除

文件仍在上传或已暂存时（发送消息前），点击任意文件标签上的 **X** 即可移除该文件。消息一经发送，其附件即为永久内容，无法移除。

## 附件如何提供给智能体

当你发送带有附件的消息时，文件会：

1. **存储** 在 CubePlex 的对象存储中。
2. **注入** 到智能体沙箱中的已知路径，使其可通过代码执行工具访问。
3. 在消息上下文中被 **引用**，让智能体知道有哪些可用文件及其类型。

对于图片，智能体还可以通过支持视觉功能的模型直接“看到”图片。

## 在聊天中查看附件

已发送的附件会显示在消息气泡上方：

- **图片** 显示为缩略图。点击缩略图可打开全尺寸灯箱预览。
- **文档和其他文件** 显示为包含文件名和大小的紧凑标签。点击标签可在侧面板中打开预览：
  - PDF、Markdown、纯文本、代码、JSON 和 CSV 文件会直接渲染。
  - Word、Excel 和 PowerPoint 文件（`.docx` / `.xlsx` / `.pptx`）通过 Microsoft Office Online 查看器渲染。部署必须配置公开 URL（`api.public_url`）；查看器会通过短期有效的令牌链接获取文档。
  - 音频和视频文件会以播放控件打开。
  - 无法预览的文件类型（压缩包、未知格式）则会显示下载按钮。

## 大小和配额限制 {#size-and-quota-limits}

| 限制 | 默认值 |
|---|---|
| **最大文件大小** | 每个文件 50 MB |
| **每条消息最多文件数** | 单条消息可引用 10 个附件 |
| **每个对话的最大值** | 所有附件合计 500 MB |

这些限制可由组织管理员配置。如果超过单个文件限制，上传会被拒绝。如果附加的文件数超过每条消息上限，发送消息会被拒绝——请将文件拆分到多条消息中。如果超过每个对话的配额，你需要开始一个新对话，或请管理员调整限制。

## 提示

- **尽早附加上下文文档。** 在第一条消息中上传参考资料（规范、数据文件、风格指南），让智能体从一开始就拥有完整上下文。
- **将图片与支持视觉功能的模型配合使用。** 如果需要智能体理解截图、图表或照片，请确保所选模型支持视觉功能。参阅[模型选择](./model-selection.md)。
- **结合文本和文件。** 始终附上文字消息，说明你希望智能体如何处理附加文件。仅上传文件而没有说明，得到的结果会更不实用。
- **上传前检查文件大小。** 大文件（尤其是图片）可能拖慢上传速度。如果要分享代码或数据，可以考虑将多个文件压缩为单个归档文件。
