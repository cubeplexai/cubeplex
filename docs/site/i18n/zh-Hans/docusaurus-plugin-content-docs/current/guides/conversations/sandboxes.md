# 管理沙箱

沙箱是智能体为对话或话题运行代码的隔离环境。每个沙箱都有自己的文件和已安装软件包，CubePlex 会在容器重启后保留它——因此，下次打开对话时，其中的工作文件仍然存在。工作区设置中的 **沙箱** 标签会显示当前工作区中属于你的所有沙箱，并允许你在出现问题时对其采取操作。

## 打开沙箱标签

进入工作区并打开 **设置** → **沙箱**。列表仅显示当前工作区中属于你的沙箱。如需查看所有成员的沙箱，组织管理员应使用管理员级别的沙箱可观测性视图（参阅[沙箱管理](../../admin/sandbox.md)）。

## 每一行显示的内容

无论容器当前是否正在运行，你拥有的每个活动沙箱都会有一行：

- **状态** — 显示沙箱运行时状态的徽章（见下文）。
- **范围标签** — 该沙箱属于哪个对话或话题：
  - **你的工作区沙箱** — 你的个人沙箱，用于不属于话题的一对一对话。
  - **群组聊天：`{title}`** — 独立的群组对话（不属于话题）。
  - **话题：`{title}`** — 属于某个[话题](./topics.md)的沙箱。
  - **（已删除）** — 拥有此沙箱的对话或话题已被删除。沙箱行仍会保留，以便你进行清理。
- **上次活动时间** — 智能体上次在其中运行代码的时间。
- **重启** 和 **删除** — 两种可用操作（见下文）。

状态为 **关闭**（容器已停止）的沙箱仍会显示在列表中。这是有意设计：沙箱的文件仍在磁盘上，下次你在其对话中发送消息时，它会再次启动。聊天前无需手动重启它。

## 重启

**重启** 会停止沙箱的容器，但保留沙箱行及其所有文件。状态会变为 **关闭**；下一次智能体需要在该对话中运行代码时，一个全新的容器会在同一存储上启动，且所有文件均保持完好。

当容器处于异常状态——进程卡住、shell 损坏，或你希望中止失控命令——但仍想保留工作文件时，请使用重启。

## 删除

**删除** 会永久移除沙箱。该行会被软删除，容器会停止，并且该对话或话题的沙箱不会再次启动。下次你在那里发送消息时，CubePlex 会预配一个存储为空的全新沙箱。

:::caution
删除无法撤销。已存储的文件会保留在磁盘上，供你的运维人员回收（CubePlex 无法直接删除底层存储）。如果你只想获得一个全新的容器，同时保留文件，请改用 **重启**。
:::

## 沙箱状态

每一行的徽章反映沙箱的运行时状态：

| 徽章 | 含义 |
|---|---|
| **运行中** | 容器已启动，可随时执行代码。 |
| **正在启动** | 正在预配容器。这是暂时状态，很快会变为运行中。 |
| **已暂停** | 容器已暂停（长时间空闲）。下次使用时会自动恢复。 |
| **正在暂停** / **正在恢复** | 暂停或恢复过程中的过渡状态。 |
| **正在停止** | 容器正在因重启或删除操作而停止。 |
| **关闭** | 容器已停止，但沙箱行及其文件仍然存在。下次使用时会再次启动。 |
| **失败** | 最近一次预配尝试失败。使用 **重启** 重试，或使用 **删除** 清除它。 |

## 存储隔离

每个沙箱都拥有独立存储——一个沙箱中的文件永远不会对其他沙箱可见。此规则同样适用于[话题中的共享沙箱](./topics.md)：使用 **专用话题沙箱** 模式的话题会获得一个拥有独立存储的新沙箱，与创建者的个人沙箱和所有其他话题相隔离。从中升级的对话中的文件 **不会** 被带入专用话题沙箱。

## 沙箱何时出现和消失

- 当智能体首次在尚无沙箱的对话或话题中运行代码时，会创建一个沙箱行。
- 该行会保留在列表中，直到你将其 **删除**（或者其所属对话/话题被删除后，你清理该孤立行）。
- 停止、暂停或重启容器 **不会** 移除该行——只有删除会移除。

## 提示

- **删除前先重启。** 如果沙箱行为异常但你想保留文件，请重启它。仅在确实想要从头开始时才使用删除。
- **清理孤立行。** 如果某行范围显示为 **（已删除）**，说明拥有它的对话或话题已不存在。删除该行以停止为闲置容器付费。
- **你只能管理自己的沙箱。** 每位成员只能看到自己的沙箱。若要审查整个工作区的沙箱使用情况，组织管理员应使用管理员可观测性视图（参阅[沙箱管理](../../admin/sandbox.md)）。
