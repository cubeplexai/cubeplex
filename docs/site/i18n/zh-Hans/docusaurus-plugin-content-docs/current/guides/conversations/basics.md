# 对话基础

对话是你与 AI 智能体交互的地方。每个对话都是限定在当前工作区内的持久消息线程，并会在多个轮次间保留完整上下文。

## 开始对话

打开工作区后，你会进入带有空输入栏的主页。输入消息并按 **Enter**（或点击发送按钮）即可开始新对话。CubePlex 会自动创建对话并将你带到该对话页面。

你也可以随时点击侧边栏中的 **新建聊天** 来开始全新的对话。

## 对话界面

界面主要分为三个区域：

- **侧边栏**（左侧）— 列出最近的对话、工作区导航项（技能、工具、记忆、设置）以及工作区切换器。
- **聊天区域**（中间）— 消息线程和输入栏。
- **侧面板**（右侧，按需打开）— 预览制品、附件图片、工具调用详情或沙箱浏览器。

![包含聊天线程和制品预览面板的对话视图](/img/conversations/conversation-layout.png)

输入栏位于底部，包含：

- **模型预设选择器** — 选择用于发送此消息的模型。
- **思考控制项** — 设置推理深度（关闭、低、中、高、最大）。
- **附加按钮**（回形针图标）— 向消息添加文件。
- **文本区域** — 输入消息。按 **Enter** 发送，按 **Shift+Enter** 换行。
- **发送/停止按钮** — 发送消息或停止正在进行的回复。

## 斜杠命令

在输入框**开头**输入 **`/`**（整段草稿仅为该命令 token）可打开可筛选的命令面板。用 ↑/↓ 高亮，**Enter** 或 **Tab** 执行，**Esc** 关闭。客户端命令不会把斜杠文本当作用户消息发送。

| 命令 | 作用 |
| --- | --- |
| `/help` | 在面板中列出可用命令 |
| `/new` | 开始新对话（等同侧边栏「新对话」） |
| `/stop` | 取消进行中的运行（仅流式中可见） |
| `/model` | 打开模型 / 推理强度选择器 |
| `/effort` | 打开思考 / 推理强度控制 |
| `/rename` | 在侧边栏重命名当前对话 |
| `/share` | 打开分享面板 |
| `/attach` | 打开文件附件选择 |
| `/skills` | 打开工作区技能页 |
| `/mcp` | 打开工作区 MCP 连接器 |
| `/compact` | 强制压缩当前对话上下文（仅空闲时） |

无匹配的文本（如 `/zzzzz`）会按普通消息发送（流式中则引导）。在 `/` 后输入空格可写字面量前导斜杠。

## 智能体可以做什么

在对话过程中，智能体可以：

- **以文本回复** — 提供使用 Markdown 格式的常规对话回复。
- **思考** — 在可折叠的“思考”块中展示推理过程（启用思考时）。
- **调用工具** — 调用 MCP 连接器以访问外部服务（数据库、API、SaaS 产品）。
- **执行代码** — 在沙箱环境中运行命令。
- **生成制品** — 生成可下载文件、实时网站预览、图片、数据文件等。参阅[制品](./artifacts.md)。
- **保存记忆** — 存储供未来对话使用的事实、偏好或决策。参阅[记忆概览](../memory/overview.md)。

## 引导和停止

智能体正在回复时，你可以：

- **停止回复** — 点击停止按钮（方形图标）取消当前回复。
- **中途引导** — 在智能体仍在回复时输入消息并按 Enter。这会发送一条“引导”指令，无需等待智能体完成即可改变其处理方向。

当智能体需要你提供输入时（例如继续前的确认），输入栏会锁定，直到你回复聊天中出现的提示卡片。

## 管理对话

在侧边栏中右键点击任意对话（或点击三点菜单），可使用以下操作：

- **重命名** — 为对话设置描述性标题。默认情况下，CubePlex 会根据第一条消息自动生成标题。
- **置顶** — 置顶的对话会固定在侧边栏列表顶部，便于快速找到。
- **取消置顶** — 将对话从置顶区域移除。
- **删除** — 软删除对话。它会从侧边栏消失，但其数据（消息、制品、成本记录）会在内部保留。

## 自动生成的标题

当你在对话中发送第一条消息时，CubePlex 会使用 LLM 生成一个概括主题的简短标题。如果你希望使用自定义标题，请从侧边栏菜单中重命名该对话。

## 使用先前的工作

智能体可以搜索当前工作区中的对话，并查看你有权访问的制品。默认情况下，它会读取对话中最近几轮的一个小窗口，以便快速理解相关上下文。必要时，它还可以检索特定结果的详细历史工具输出。

此访问为只读：智能体无法通过该功能删除制品。

## 提示

- **为活跃项目使用置顶。** 置顶你经常返回的对话，这样无论开始多少新聊天，它们都会保持可见。
- **每个对话只讨论一个主题。** 智能体会将整个对话历史作为上下文。在同一个线程中混入无关主题会稀释上下文，并可能产生相关性较低的回复。
- **使用引导而不是停止。** 如果智能体正朝错误方向处理，请使用后续消息引导它，而不是停止后重新开始。这样可以保留智能体已经建立的上下文。
- **查看思考块。** 启用推理后，展开思考块即可了解智能体如何得出答案。这对复杂任务尤其有用。
