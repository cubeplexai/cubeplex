# Search MCP Sources + Citation Rules Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans to implement this plan task-by-task.

**Date:** 2026-05-27 (revised 2026-05-28)
**Goal:** Ship hosted search MCP connectors (Tavily, Exa, Jina) with correct `tool_citation_defaults`, extend the runtime so non-Bearer static auth (custom header, URL query-param) works, and bake real captured response shapes into unit tests so citation extraction is regression-guarded against silent provider-side schema changes. Bocha and Perplexity are deferred (no usable hosted MCP at PR time).

**Architecture:** Two surfaces change.
1. **Auth plumbing (new).** `MCPConnectorTemplate` and `MCPConnectorInstall` gain three columns — `static_auth_style`, `static_auth_header_name`, `static_auth_query_param`. The runtime helper renames to `_resolve_auth_from_spec` and returns `(headers, server_url)`, with the URL rewritten for query-param style. Pure helpers `_apply_static_credential` + `_inject_query_param` make the dispatch trivially unit-testable.
2. **Catalog + tests (config).** Three search entries seed alongside `webtools` with the right auth shape and citation mappings. `CitationConfig.extract_items` learns to walk a dotted `content_field` so Bocha (and any future nested-shape provider) lights up without code changes when it ships. Captured response fixtures in `tests/unit/fixtures/search_responses/` drive both always-on unit tests and opt-in live-API smoke tests.

**Tech Stack:** Python 3.13, FastAPI, SQLModel/SQLAlchemy async, Pydantic, pytest, Alembic (one autogenerated migration: `49844ea5ac7b_mcp_template_install_static_auth_style_.py`).

## Decisions locked

- **5 providers requested, 3 ship in v1.** Bocha and Perplexity have no usable hosted MCP at PR time — Bocha never published one, Perplexity's CTO moved away from MCP. Both stay in spec Open Questions; Bocha's REST shape is captured as a fixture so the day a hosted MCP appears the catalog seed is one PR.
- **Auth shape is data, not branches in code.** Three columns + a pure dispatch helper in `cubepi_runtime.py`. No connector-specific `if slug == "exa": ...` anywhere.
- **No flaky visible-marker E2E.** Model adherence to `【N-M】` rules in the final answer text is unreliable across providers; turning that into a green/red CI assertion gates merges on the model's whims. The citation side channel + captured-structure unit tests + prompt-cache E2E gate are the supported regression surface. The user explicitly dropped the marker-in-text E2E from scope.
- **Header/param names are configurable.** Not hardcoded; come from the template/install row, so a workspace can rotate `x-api-key` to `X-Lark-MCP-UAT` for a custom server without code changes.
- **Default credential policy stays `org`** to match the existing `webtools` entry — keys are shared at the org level. A per-workspace policy for metered search keys is OQ-E.
